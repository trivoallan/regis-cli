# Post-install instructions for {{ cookiecutter.project_name }}

## Setup

1. Install dependencies:

   ```bash
   pnpm install
   ```

2. A first analysis of `ghcr.io/trivoallan/regis-cli` was run automatically during bootstrap.
   To add more reports:

   ```bash
   regis-cli analyze <image> --archive ./static/archive
   ```

3. Start the dev server:
   ```bash
   pnpm dev
   ```

{% if cookiecutter.platform == "github" -%}

## GitHub Setup

1. **GitHub Pages** is automatically configured to deploy via **GitHub Actions**.
   - Deployment occurs automatically on every push to `main`.
2. **Pull Request Previews**:
   - To enable automated previews for new reports, you MUST create a **Personal Access Token (PAT)**.
   - Go to **Settings → Developer settings → Personal access tokens → Tokens (classic)**.
   - Generate a new token with `repo` and `workflow` scopes.
   - Add this token as a **Repository Secret** named `REGIS_BOT_TOKEN` in your archive repository.
3. **Trigger Analysis**:
   - Manually via **Actions → Analyze Image → Run workflow**.
   - This will create a **Pull Request** for you to review (with a live preview if `REGIS_BOT_TOKEN` is set).
   - Once merged into `main`, the report is permanently added to the archive.
     {%- elif cookiecutter.platform == "gitlab" -%}

2. Push commits to `main` — the pipeline rebuilds and deploys whenever `static/archive/**` changes.

## Adding reports

From any project using regis-cli, archive reports directly into this repository:

```bash
# Clone this archive repo alongside your project
regis-cli analyze <image> \
  --archive /path/to/{{ cookiecutter.project_slug }}/static/archive

# Then commit and push in this repo
cd /path/to/{{ cookiecutter.project_slug }}
git add static/archive/
git commit -m "chore: add report for <image>"
git push
```

Or add an existing `report.json`:

```bash
regis-cli archive add ./report.json \
  --archive-dir /path/to/{{ cookiecutter.project_slug }}/static/archive
```
