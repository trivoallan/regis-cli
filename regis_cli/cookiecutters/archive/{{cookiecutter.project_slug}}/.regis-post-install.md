# Post-install instructions for {{ cookiecutter.project_name }}

## Setup

1. Install dependencies:

   ```bash
   pnpm install
   ```

2. Add your first report to the archive:

   ```bash
   regis-cli analyze <image> --archive ./static/archive
   ```

3. Start the dev server:
   ```bash
   pnpm dev
   ```

1. Enable **GitHub Pages** in your repository settings (source: GitHub Actions).
2. Push commits to `main` — the workflow rebuilds and deploys the site automatically whenever `static/archive/**` changes.

## Adding reports

From any project using regis-cli, archive reports directly into this repository:

```bash
# Clone this archive repo alongside your project
regis-cli analyze <image> \
  --archive /path/to/{{ cookiecutter.project_slug }}/static/archive

# Then commit and push in this repo
cd /path/to/{{ cookiecutter.project_slug }}
git add static/archive/
git commit -m "chore: add report for <image>"
git push
```

Or add an existing `report.json`:

```bash
regis-cli archive add ./report.json \
  --archive-dir /path/to/{{ cookiecutter.project_slug }}/static/archive
```
